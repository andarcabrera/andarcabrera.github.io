Ruby Classes vs. JavaScript Objects

July 10, 2015

Classes in Ruby and constructor functions in Java Script are similar in that they allow us to build objects that share similar properties. Instead of creating objects one by one, which would be very time consuming, we can create a class in Ruby and then create various instances of this class or, in JavaScript, we can use the constructor function. Below is an example:

Ruby Class

  • class Book
    attr_reader :title, :author
    attr_accessor :price
    def initialize (title, author, price)
      @title = title
      @author = author
      @price = price
    end

    gone_with_the_wind = Book.new("Gone with the Wind", "Margaret Mitchell", $20")

JavaScript Constructor function

  • function Book(name, author, price) {
      this.name = name;
      this.author = author;
      this.price = price
    }
    goneWithTheWind = new Book("Gone with the Wind", "Margaret Mitchell", 30);

A main difference is that classes in Ruby are objects in their own right and they are all subclasses of the Class class, while the constructor in JavaScript is a function. Objects in Ruby can inherit a lot of behaviors from their superclasses, while in JavaScript, the shared properties will be the ones defined in the constructor functions or the ones that are subsequently added.
